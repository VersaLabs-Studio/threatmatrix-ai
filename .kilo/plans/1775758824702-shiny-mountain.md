# Plan: Fix Severity Distribution — Only MEDIUM Alerts Being Generated

## Root Cause Analysis

**Symptom**: All 1,482 alerts are MEDIUM severity (0.50-0.52 composite score). ZERO critical, high, low, or info.

**Why this happens:**

The ensemble scoring pipeline:
```
composite = 0.30 × IF_score + 0.45 × RF_attack_conf + 0.25 × AE_score
```

Where:
- `IF_score` = Isolation Forest anomaly score (contamination=0.10)
- `RF_attack_conf` = `1.0 - P(normal)` from Random Forest
- `AE_score` = Autoencoder normalized reconstruction error

The models were trained on **NSL-KDD** data which has a very different feature distribution than live internet traffic. When live flows pass through:

1. **RF model** classifies most live traffic as "normal" with high confidence → `RF_attack_conf` is low (0.1-0.3)
2. **IF model** flags ~10% as anomalous (contamination=0.10) but with moderate scores (0.5-0.7)
3. **AE model** has moderate reconstruction error for live traffic (0.4-0.6)

The weighted average consistently lands at **0.50-0.52**, just barely crossing the MEDIUM threshold (0.50).

**Current thresholds:**
```python
critical: 0.90  # Requires all 3 models to strongly agree
high:     0.75  # Requires 2+ models to strongly agree
medium:   0.50  # Barely anomalous
low:      0.30  # Slightly unusual
```

**The fix**: Calibrate thresholds to match the actual score distribution of live traffic, not theoretical NSL-KDD scores.

## Implementation Plan

### Step 1: Add Per-Model Score Logging to Worker

Add logging to see exactly what each model outputs for anomalous flows. This will tell us the actual score distribution.

**File**: `backend/ml/inference/worker.py`

In `_process_message()`, after scoring but before the existing batch stats log:
```python
# Log per-model scores for anomalous flows
if is_anomaly:
    logger.info(
        "[Worker] Flow %s scores: IF=%.3f RF=%.3f AE=%.3f composite=%.3f severity=%s",
        flow_id,
        result["if_score"],
        1.0 - rf_pred.get("class_probabilities", {}).get("normal", 1.0) if rf_pred.get("is_anomaly") else 0.0,
        result["ae_score"],
        result["composite_score"],
        severity,
    )
```

### Step 2: Calibrate Thresholds

Based on the score distribution (all alerts at 0.50-0.52), set thresholds that spread alerts across severity levels:

**Proposed thresholds:**
```python
ALERT_THRESHOLDS = {
    "critical": 0.65,  # Top 5% of anomalous flows
    "high":     0.55,  # Top 15%
    "medium":   0.45,  # Top 30%
    "low":      0.35,  # Top 50%
}
```

**File**: `backend/ml/training/hyperparams.py`

## Why This Is Critical

Without diverse severity levels:
- Demo looks broken (only one color badge)
- No critical/high alerts to showcase the system's detection capability
- AI Analyst can't demonstrate severity-based prioritization
- Toast notifications all look the same
- Alert Console has no severity filtering to demonstrate

## Expected Outcome

After threshold calibration:
- ~5% of alerts: CRITICAL
- ~15% of alerts: HIGH
- ~30% of alerts: MEDIUM
- ~30% of alerts: LOW
- ~20% of alerts: INFO (benign but flagged)

This matches what a real SOC would see — most alerts are low/medium, a few are critical.
