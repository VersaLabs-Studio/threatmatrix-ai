## Part 1: Worker Latency Logging

**File:** `backend/ml/inference/worker.py`

Add per-model timing breakdown to `_process_message()`:

```python
async def _process_message(self, data: str) -> None:
    start = time.time()
    msg = json.loads(data)
    # ... parse ...
    
    # Time preprocessing
    X = self.preprocessor.preprocess_flow(features)
    t_preprocess = time.time()
    X_batch = X.reshape(1, -1)
    
    # Time per-model scoring (break out model_manager.score_flows)
    if_scores = self.model_manager.if_model.score(X_batch)
    t_if = time.time()
    rf_preds = self.model_manager.rf_model.predict_with_confidence(X_batch)
    t_rf = time.time()
    ae_scores = self.model_manager.ae_model.score(X_batch)
    t_ae = time.time()
    
    results = self.model_manager.scorer.classify(if_scores, rf_preds, ae_scores)
    t_ensemble = time.time()
    
    result = results[0]
    
    # Log per-flow latency breakdown
    logger.info(
        "[Worker] Flow %s latency: preprocess=%.1fms IF=%.1fms RF=%.1fms AE=%.1fms ensemble=%.1fms total=%.1fms",
        flow_id,
        (t_preprocess - start) * 1000,
        (t_if - t_preprocess) * 1000,
        (t_rf - t_if) * 1000,
        (t_ae - t_rf) * 1000,
        (t_ensemble - t_ae) * 1000,
        (t_ensemble - start) * 1000,
    )
    
    # On alert: also publish latency to ml:metrics
    if is_anomaly and severity in ("critical", "high", "medium"):
        t_alert = time.time()
        await self._publisher.publish("ml:metrics", json.dumps({
            "event": "latency",
            "payload": {
                "flow_id": flow_id,
                "preprocess_ms": round((t_preprocess - start) * 1000, 2),
                "if_ms": round((t_if - t_preprocess) * 1000, 2),
                "rf_ms": round((t_rf - t_if) * 1000, 2),
                "ae_ms": round((t_ae - t_rf) * 1000, 2),
                "total_ms": round((t_alert - start) * 1000, 2),
                "timestamp": datetime.now(timezone.utc).isoformat(),
            }
        }))
```

**Why break out models individually:** `model_manager.score_flows()` runs all 3 models sequentially in one call. We need to see IF vs RF vs AE timing individually to identify bottlenecks.

### 3b. Toast Redesign (NotificationToast.tsx)

**Layout per toast:**
```
┌─────────────────────────────────────────────────┐
│ [ICON] SEVERITY TITLE          [SCORE%]    [×]  │
│ Source IP → Dest IP         MODEL AGREEMENT      │
│ ── score bar (filled to score%) ─────────────── │
│ IF: 0.91 │ RF: 0.88 │ AE: 0.85                 │
└─────────────────────────────────────────────────┘
```

**Severity config:**
| Severity | Icon | Color | Auto-dismiss |
|----------|------|-------|-------------|
| critical | AlertTriangle (pulsing) | red | 10s |
| high | ShieldAlert | orange | 8s |
| medium | Shield | yellow | 6s |
| low | Info | blue | 5s |
| info | Bell | gray | 4s |

**Visual emphasis for critical:**
- Pulsing border animation (`severity-breathe`)
- Brighter glow shadow
- Longer duration (10s)
- Click navigates to `/alerts`

**Score bar:**
- Horizontal bar filled to `composite_score * 100%`
- Color matches severity
- Smooth fill animation
