# Plan: WebSocket Verification Fix + Correct Architecture Approach

## The Problem

The previous agent tried to **replace REST polling with WebSocket as the primary data source** by:
1. Adding event accumulation arrays to `useWebSocket` hook
2. Creating `useFlowsFromWS` / `useAlertsFromWS` hooks
3. Switching War Room and Alerts page to consume WS-only data
4. Bypassing token validation in DEV_MODE

**This failed because:**
- WS connection never established (token missing in DEV_MODE, CORS issues)
- Even with CORS fixed, WS events were zero (connection not connecting)
- Removing REST polling without working WS meant zero data on the page
- The approach treated WS as a full data replacement, not a real-time supplement

## What the Master Doc Actually Says

The architecture doc (Part 2, Section 6) defines:
```
Capture Engine â”€â”€â–ş Redis Pub/Sub â”€â”€â–ş FastAPI â”€â”€â–ş WebSocket â”€â”€â–ş Browser
                   (flows:live)      (subscriber)  (broadcast)  (N clients)
```

This is the **real-time event delivery pipeline**. It does NOT mean:
- REST APIs should be removed
- WS should replace the flow/stats/alerts REST endpoints
- Initial data load should come from WS

**It means:**
- REST APIs provide **baseline data, CRUD operations, stats, and initial load**
- WebSocket provides **real-time push** of NEW events as they happen
- Components should supplement REST data with WS events

## What Actually Needs to Be Done

### Step 1: Fix the WebSocket Connection (Dev Environment)

The WebSocket won't connect because:
1. No token in localStorage (no login flow completed)
2. CORS doesn't include `http://localhost:3001` (current dev port)
3. Backend token validation always requires JWT, even in DEV_MODE

**Fix (minimal, conservative):**
- `backend/app/config.py`: Add `http://localhost:3001` to CORS_ORIGINS
- `backend/app/api/v1/websocket.py`: In `validate_token()`, return dev user in DEV_MODE so WS connects without a real token
- `backend/app/api/v1/websocket.py`: Make token query parameter optional (`token: str | None`)

### Step 2: Fix the WebSocket Client Reconnection Bug

`frontend/lib/websocket.ts` line 64 references `this.subscribedChannels.size` but nothing ever adds channels to this set. On reconnect, subscriptions are lost.

**Fix:**
- In `subscribe()`, add channel to `this.subscribedChannels` set
- In `unsubscribe()`, remove channel from `this.subscribedChannels` set
- In `_open()`, re-subscribe to all channels on reconnect

### Step 3: Fix the Heartbeat Gap

The hook polls `wsClient.isConnected` every 30s but never sends an actual ping to the backend.

**Fix:**
- In `frontend/lib/websocket.ts`, send `{"action":"ping"}` in the heartbeat interval to keep connection alive
