#### Subtask 4.1: Detection Latency Logging

**File:** `backend/ml/inference/worker.py`  
**Current State:** Worker logs only aggregate stats (flows_scored, anomalies_detected, alerts_created, avg_inference_ms)

**Requirements (per Master Doc Part 4 §8):**
1. Add timestamp when flow is received from Redis (`t_flow_received`)
2. Add timestamp when ML scoring completes (`t_scored`)
3. Add timestamp when alert is created (`t_alert_created`)
4. Calculate and log end-to-end latency: `t_alert_created - t_flow_received`
5. Publish latency metrics to Redis channel `ml:metrics` for frontend display (War Room)

**Implementation Sketch:**
```python
# In _process_message():
start = time.time()
# ... existing preprocessing and scoring ...
t_scored = time.time()

# In _create_alert():
t_alert_created = time.time()
latency_ms = (t_alert_created - start) * 1000  # total pipeline latency

# Log per-alert latency
logger.info("[Worker] Alert created in %.1fms (IF:%.1f, RF:%.1f, AE:%.1f, publish:%.1f)",
    latency_ms, if_time_ms, rf_time_ms, ae_time_ms, publish_time_ms)

# Publish to Redis for real-time metrics
await self._publisher.publish("ml:metrics", json.dumps({
    "event": "latency",
    "payload": {
        "flow_id": flow_id,
        "latency_ms": round(latency_ms, 2),
        "inference_ms": round((t_scored - start) * 1000, 2),
        "timestamp": datetime.now(timezone.utc).isoformat(),
    }
}))
```

**Success Criteria:**
- Latency values logged in worker output (visible via `docker compose logs -f ml-worker`)
- `ml:metrics` channel receives messages (verify with `redis-cli PUBSUB CHANNELS`)
- Frontend War Room can subscribe and display rolling average latency (optional, backend focus for now)
