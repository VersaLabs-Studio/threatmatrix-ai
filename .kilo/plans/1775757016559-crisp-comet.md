## 🔍 Root Cause Analysis

### Issue 1: Sidebar Toggle Button Not Visible

**Location:** `frontend/app/ai-analyst/page.tsx:319-348`

**Root Cause:** The toggle button uses `position: 'absolute'` but its parent container (line 313) has `display: flex` without `position: relative`. This causes the button to be positioned relative to the nearest positioned ancestor (likely the page/AuthGuard), placing it at `left: 8, top: 16` relative to the page top-left corner, NOT the sidebar area. The button may be rendered behind the main content due to z-index stacking.

**Fix:** 
1. Add `position: relative` to the main container (line 313)
2. Ensure z-index layering: main container should not overlap the toggle button
3. Or restructure so toggle is inside the header area instead of outside the main layout

### Issue 2: Conversation Messages Not Restored on Session Switch

**Location:** `frontend/app/ai-analyst/page.tsx:242-256`

**Root Cause:** The `useChatHistory` hook saves messages correctly via `updateSession()`, but there's NO code to RESTORE messages from a session back into the `useLLM` hook when switching sessions. The `useEffect` at line 242-248 only clears messages, it doesn't load them.

**Current flow:**
- Switch session → `clearMessages()` → chat is cleared but empty
- New session has `messages: []` from creation
- No code calls anything like `loadMessagesIntoLLM(session.messages)`

**Fix:**
- `useChatHistory` should expose a `loadSessionMessages` function that can set messages in `useLLM`
- Or restructure so `useLLM` is aware of the active session and loads messages from it

### Issue 3: Missing Message Loading Mechanism

The `useLLM` hook doesn't expose `setMessages`, so there's no way to restore messages from a session. Options:

1. Expose `setMessages` from `useLLM` (simple but breaks encapsulation)
2. Pass messages through the session and have `useLLM` accept initial messages
3. Use a shared state manager (context) to coordinate between hooks
