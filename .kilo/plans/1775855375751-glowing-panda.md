# Amharic/English i18n Implementation Plan

## Overview
Implement comprehensive internationalization (i18n) for the entire ThreatMatrix AI frontend, supporting English and Amharic languages. Replace all hardcoded UI strings with translation keys, ensuring every user-facing text is localizable.

## Current State Analysis
- Custom `useTranslation` hook exists with basic translations
- Message files (`en.json`, `am.json`) contain limited translations
- Language toggle in TopBar with localStorage persistence
- Extensive hardcoded strings throughout components

## Implementation Steps

### 1. Migration to next-intl (Optional but Recommended)
- Install and configure next-intl for better integration with Next.js
- Replace custom hook with next-intl's useTranslations
- Update next.config.ts with i18n configuration
- Migrate existing translations to next-intl format

### 2. Comprehensive Translation File Expansion
- Audit all components for hardcoded strings using grep search
- Categorize strings by feature areas:
  - Navigation/Sidebar
  - Dashboard/War Room
  - Alerts/Console
  - AI Analyst
  - Admin/ML Ops
  - Forms/Inputs
  - Status Messages
  - Error Messages
  - Tooltips/Labels
- Add all identified strings to en.json and am.json
- Include severity levels: Critical/High/Medium/Low translations

### 3. Component Refactoring
- Replace hardcoded strings with t() calls in all components:
  - Layout components (TopBar, Sidebar, StatusBar)
  - Page components (war-room, alerts, ai-analyst, etc.)
  - Shared components (buttons, forms, tables)
  - Hooks and utilities
- Update dynamic content rendering to use translations
- Handle pluralization and parameterized strings

### 4. Persistence and State Management
- Ensure language preference persists across sessions
- Handle language switching in real-time without page reload
- Update language detection logic if needed

### 5. Testing and Validation
- Verify all UI elements display translated text
- Test language switching functionality
- Validate Amharic text rendering (RTL considerations if needed)
- Check console logs and error messages

## Key Challenges
- Identifying every hardcoded string (338+ matches found in initial grep)
- Maintaining translation consistency across components
- Handling dynamic content with translations
- Ensuring Amharic translations are accurate and contextually appropriate

## Acceptance Criteria
- [ ] Language toggle button in TopBar functional
- [ ] All navigation items translated (War Room → የጦርነት ክፍል, etc.)
- [ ] Severity badges show translated text (Critical → ወሳኝ, High → ከፍተኛ, etc.)
- [ ] Every user-facing string is translatable
- [ ] Language preference persists in localStorage
- [ ] No hardcoded English strings remain in UI

## Estimated Effort
- Translation audit: 2-3 hours
- File expansion: 4-6 hours
- Component refactoring: 8-12 hours
- Testing: 2-4 hours
- Total: 16-25 hours

- next-intl package (already in dependencies)
