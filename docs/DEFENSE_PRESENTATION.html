ML Architecture

Why Three Models?

Each model detects fundamentally different threat categories

🌲

Isolation Forest

Unsupervised

The Sentinel — catches zero-day attacks. Isolates anomalies through random tree splits. No labeled data needed.

🎯

Random Forest

Supervised

The Classifier — classifies into 5 NSL-KDD categories with explainable feature importance.

🧬

Autoencoder

Deep Learning

The Pattern Learner — neural net trained on normal only. High reconstruction error = anomaly.

Ensemble = Robustness. If one model false-fires, the other two compensate. Not redundancy — deliberate architectural design.

Ensemble Scoring

The Mathematics

composite = 0.30 × IFscore + 0.45 × RFconfidence + 0.25 × AErecon_error
0.30

Isolation Forest — zero-day detection capability

0.45

Random Forest — strongest signal, explicit labels

0.25

Autoencoder — deep pattern recognition

Severity Thresholds

CRITICAL≥ 0.90
HIGH≥ 0.75
MEDIUM≥ 0.50
LOW≥ 0.30

Weight Rationale

Determined through grid search + architectural intent. RF gets highest weight due to strongest training signal. IF provides irreplaceable zero-day value. AE captures patterns tree-based models miss. Grid search converged to these approximate values.

Validation

Benchmark Results

ModelAccuracyF1 ScoreAUC-ROCInference
Isolation Forest79.68%78.75%0.9378<1ms
Random Forest74.16%—0.95761-2ms
Autoencoder60.39%—0.85172-5ms
Ensemble80.73%80.96%0.9312~5ms

AUC-ROC = 0.93 Explained

93% chance of correctly ranking a malicious flow higher than a benign one. More meaningful than accuracy — measures discrimination ability across all thresholds. Academic literature: 80-85% is state-of-the-art on NSL-KDD.

Cross-Dataset: CICIDS2017

Validated on ~2.5M flows from a completely different dataset → 83.14% accuracy. Proves generalization beyond training data. Different era, different attack types, same model.

Reflections

Challenges & Lessons Learned

Top 3 Challenges

  • 1Integration over isolation — 5 Docker containers communicating in a live, streaming, fault-tolerant pipeline. The connections between systems always break first.
  • 2LLM architecture pivot — Moving from 3 separate providers to OpenRouter mid-development. Risk that paid off: $0/month cost, better models.
  • 3NSL-KDD class imbalance — 52 U2R samples. The model wants to predict 'Normal' for everything. Class weighting helped, but honest evaluation means accepting limitations.

What We'd Do Differently

  • 1Start frontend earlier — build UI and backend in parallel from Week 1. The War Room is the demo center.
  • 2Use live traffic from Day 1 — integrating capture engine earlier = earlier real-world feedback.
  • 3Validate CICIDS2017 sooner — dual-dataset validation is a major strength, should've been Week 3 not Week 7.
  • 4Document as you build — weekly increments, not a massive batch at the end.
